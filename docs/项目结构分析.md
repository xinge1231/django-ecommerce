# Django E-commerce 项目结构分析

## 项目概述

这是一个基于 Django 2.2 构建的电商网站，支持商品浏览、购物车管理、Stripe 支付和退款功能。

## 核心组件详解

### 1. 数据模型 (core/models.py)

| 模型 | 用途 | 关键字段 |
|------|------|----------|
| `Item` | 商品信息 | title, price, discount_price, category, slug, image |
| `Order` | 订单/购物车 | user, items, shipping_address, billing_address, payment, ordered |
| `OrderItem` | 订单项 | item, quantity, ordered |
| `Address` | 收货/账单地址 | street_address, country, zip, address_type |
| `Payment` | 支付记录 | stripe_charge_id, amount, user |
| `Coupon` | 优惠券 | code, amount |
| `Refund` | 退款申请 | order, reason, email, accepted |
| `UserProfile` | 用户扩展信息 | user, stripe_customer_id, one_click_purchasing |

**模型关系图:**
```
User ──┬── UserProfile (OneToOne)
       │
       ├── Order (ForeignKey) ──┬── OrderItem (ManyToMany)
       │                        ├── Address (ForeignKey: shipping/billing)
       │                        ├── Payment (ForeignKey)
       │                        └── Coupon (ForeignKey)
       │
       └── Address (ForeignKey)
              └── CountryField
```

### 2. 视图架构 (core/views.py)

#### 类视图 (Class-Based Views)

| 视图 | 继承 | 功能 |
|------|------|------|
| `HomeView` | ListView | 首页商品列表，每页10个商品 |
| `ItemDetailView` | DetailView | 商品详情页 |
| `CheckoutView` | View | 处理结账表单，收集地址信息 |
| `PaymentView` | View | Stripe 支付处理 |
| `OrderSummaryView` | View | 购物车摘要 |
| `AddCouponView` | View | 添加优惠券 |
| `RequestRefundView` | View | 退款申请 |

#### 函数视图 (Function Views)

| 函数 | 功能 |
|------|------|
| `add_to_cart(request, slug)` | 添加商品到购物车 |
| `remove_from_cart(request, slug)` | 从购物车移除商品 |
| `remove_single_item_from_cart(request, slug)` | 减少商品数量 |
| `products(request)` | 商品列表页面 |
| `get_coupon(request, code)` | 获取优惠券 |

### 5. 业务流程

#### 购物流程图

```
1. 浏览商品 ──→ HomeView (分页展示)
       │
       ↓
2. 查看详情 ──→ ItemDetailView
       │
       ↓
3. 加入购物车 ──→ add_to_cart() → 创建/更新 Order (ordered=False)
       │
       ↓
4. 查看购物车 ──→ OrderSummaryView
       │
       ↓
5. 结账 ──→ CheckoutView (填写收货/账单地址)
       │
       ↓
6. 支付 ──→ PaymentView (Stripe API)
       │
       ↓
7. 完成 ──→ Order.ordered=True, 生成 ref_code
```

#### 订单状态流转

```
状态字段说明:
- ordered: 是否已下单
- being_delivered: 配送中
- received: 已收货
- refund_requested: 申请退款
- refund_granted: 退款已批准

状态流转:
[购物车] → [已下单] → [配送中] → [已收货]
                              │
                              ↓
                        [申请退款] → [退款成功]
```

## 设置分离策略

### 开发环境 (development.py)

```python
DEBUG = True
ALLOWED_HOSTS = ['127.0.0.1']

# SQLite 数据库
DATABASES = {
    'default': {
        'ENGINE': 'django.db.backends.sqlite3',
        'NAME': 'db.sqlite3',
    }
}

# Stripe 测试密钥
STRIPE_PUBLIC_KEY = config('STRIPE_TEST_PUBLIC_KEY')
STRIPE_SECRET_KEY = config('STRIPE_TEST_SECRET_KEY')

# 启用 debug_toolbar
INSTALLED_APPS += ['debug_toolbar']
MIDDLEWARE += ['debug_toolbar.middleware.DebugToolbarMiddleware']
```

### 生产环境 (production.py)

```python
DEBUG = False
ALLOWED_HOSTS = ['ip-address', 'www.your-website.com']

# PostgreSQL 数据库
DATABASES = {
    'default': {
        'ENGINE': 'django.db.backends.postgresql_psycopg2',
        'NAME': config('DB_NAME'),
        'USER': config('DB_USER'),
```

## 注意事项

1. **购物车持久化**: 购物车数据存储在数据库 (Order with ordered=False)，而非 Session
2. **价格计算**: OrderItem.get_final_price() 自动处理折扣逻辑
3. **Stripe 集成**: 支持保存卡片信息实现一键购买 (one_click_purchasing)
4. **地址管理**: 支持设置默认收货/账单地址
5. **优惠券**: 每个 Order 只能使用一个优惠券
